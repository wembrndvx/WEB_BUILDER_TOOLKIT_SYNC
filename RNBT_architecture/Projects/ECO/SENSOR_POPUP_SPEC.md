## 팝업 구조 개요

팝업은 2개 섹션으로 구성됩니다. (CRAC과 유사하나 상태 인디케이터 없음)

```
┌─────────────────────────────────────────────────┐
│  ● 온습도센서                               [X] │
├─────────────────────────────────────────────────┤
│                                                  │
│  ┌──────────┐  ┌──────────────────────┐         │
│  │          │  │  기본정보 테이블       │         │
│  │  센서    │  │  자산명(ID), 자산타입, │         │
│  │  이미지   │  │  용도, 제조사명, 모델, │         │
│  │          │  │  위치, 상태, 설치일    │         │
│  └──────────┘  └──────────────────────┘         │
│                                                  │
│  ① 실시간 측정값                                  │
│  ┌────────────────┐  ┌────────────────┐         │
│  │ 현재온도/적정온도 │  │ 현재습도/적정습도 │         │
│  │  28 °C / 32 °C │  │  23 % / 38 %  │         │
│  └────────────────┘  └────────────────┘         │
│                                                  │
│  온/습도 현황                                     │
│  ┌──────────────────────────────────────┐       │
│  │  ■ 온도  ● 습도                      │       │
│  │  ████████████████████ (바+라인 차트)  │       │
│  │  09시                         09시   │       │
│  └──────────────────────────────────────┘       │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 기본정보 테이블

| 필드 | 예시 값 | API 소스 | 매핑 |
|------|---------|----------|------|
| 자산명 (ID) | TH-Sensor-01 | `ast/gx` | `asset.name` |
| 자산타입 | 온습도센서 | `ast/gx` | `asset.assetType` |
| 용도 | 랙 전면 환경 감시 | `ast/gx` | `asset.usageLabel` (API 요청 완료, 필드명 미정) |
| 제조사명 | Vaisala | `vdr/g` | `vendor.name` (asset.assetModelKey → mdl/g → model.assetVendorKey → vdr/g) |
| 모델 | HMP110 | `mdl/g` | `model.name` (asset.assetModelKey → mdl/g) |
| 위치 | 2 층 전기실 | `ast/gx` | `asset.locationLabel` |
| 상태 | 정상운영 | `ast/gx` | `asset.statusType` → UI 라벨 변환 |
| 설치일 | 2025 년 3 월 23 일 | `ast/gx` | `asset.installDate` → 날짜 포맷 변환 |

### API 호출 흐름

```
1. POST /api/v1/ast/gx  { assetKey, locale: "ko" }
   → asset 기본정보 + properties

2. asset.assetModelKey가 있으면:
   POST /api/v1/mdl/g  { assetModelKey }
   → model.name (모델명), model.assetVendorKey

3. model.assetVendorKey가 있으면:
   POST /api/v1/vdr/g  { assetVendorKey }
   → vendor.name (제조사명)
```

## 온/습도 현황 (트렌드 차트)

24시간 (전일 동시간 ~ 현재시간) 데이터를 표시합니다. 온도는 바 차트, 습도는 라인 차트로 복합 차트 구성. (CRAC과 동일 형식)

### 차트 구성

| 시리즈 | 메트릭 | 차트 타입 | Y축 단위 |
|--------|--------|----------|---------|
| 온도 | `SENSOR.TEMP` | 바 차트 | °C |
| 습도 | `SENSOR.HUMIDITY` | 라인 차트 | % |

### API 호출

```
POST /api/v1/mhs/l
{
  "sort": [{ "field": "time", "direction": "ASC" }],
  "filter": {
    "assetKey": "sensor-00001",
    "interval": "1h",
    "metricCodes": ["SENSOR.TEMP", "SENSOR.HUMIDITY"],
    "timeFrom": "2026-02-02T09:00:00.000",
    "timeTo": "2026-02-03T09:00:00.000"
  },
  "statsKeys": ["avg"]
}
```

### 데이터 커버리지

| 항목 | 가능 여부 |
|------|----------|
| 온도 트렌드 | **O** (`SENSOR.TEMP`) |
| 습도 트렌드 | **O** (`SENSOR.HUMIDITY`) |
| 시계열 통계 API | **O** |

## 기획서 우측 확장 정보 (참고)

기획서 우측에는 팝업보다 더 확장된 자산 상세 정보가 표시됩니다.

| 필드 | 예시 값 | 가능 여부 |
|------|---------|----------|
| 담당자 | 홍길동 | **O** (※ `ownerUserId`로 사용자 이름 조회 API 확인 필요) |
| 설명 | 랙 전면 흡입부 센서 | **O** |
| 측정범위 | -10 ~ 60 °C / 0 ~ 95 %RH | **O** (properties.measurement_range) |
| 정확도 | ±0.3 °C / ±2 %RH | **O** (properties.accuracy) |
| 설치 위치 유형 | 랙 전면 흡입부 | **O** (properties.install_position_type) |

> **참고**: 기획서 우측 상단에 [온습도 현황] [온습도 분포도] 탭이 있음. 분포도 탭은 별도 화면으로 이 팝업 스펙 범위 외.
